# Test_MD_Skill_Webitebuilder

## Projektziel
Sandbox zum Lernen und Vergleichen:
- Claude Skills und Agenten vs. Standard-Claude-Funktionen
- Organisation von .md-Dateien und Skills über Projekte hinweg
- GitHub-Repos in VS Code Console verwalten

## Kontext
- Kein Produktivcode — alles ist Experiment und Lernmaterial
- Ergebnisse jedes Experiments werden in `experiments/` dokumentiert
- Vergleich ist immer: Standard → Skill → Agent

## Projektstruktur
```
website/        ← Testwebseite (HTML/CSS/JS)
experiments/    ← Ein Unterordner pro Experiment
docs/           ← Erkenntnisse, Vergleiche, Notizen
.claude/
  commands/     ← Projekt-spezifische Skills (Slash-Commands)
```

## Regeln für Claude in diesem Projekt
- Immer erklären WARUM ein Ansatz gewählt wurde
- Bei jedem Experiment dokumentieren: Was war der Unterschied zur Standardfunktion?
- Kurz und direkt — kein unnötiger Boilerplate-Code
